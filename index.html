ПК "Робот-загрузчик и архив документов" 

Программный комплекс для бухгалтеров, складских операторов, кадровиков способный распознавать пачки(до сотен листов за раз)любых сканированных документов. До 5 способов распознавания одновременно, из них 2 бесплатных, локальных OCR,1 бесплатный и 2 платных онлайн OCR. "Выжимает" максимум качества загрузки документов из бесплатных OCR и может быть в РАЗЫ дешевле конкурентов. Работает без Интернета. Поддерживает любые форматы изображений(BMP,JPG и т.п.) в том числе в контейнере PDF. Автоматическое создание Контрагентов и № ГТД. Полуавтоматическое создание "Номенклатуры поставщика" и "Номенклатуры". Готовые правила для УПД,ТОРГ12,СФ,счет и акт по форме 1С. Без описания правил, любые документы могут загружаться частично(В шапке Контрагент, Организация. В таблице возможно Номенклатура ,количество,
цена,сумма.) Поддерживаются БП3.0(2.0),УТ10.3-11,КА1.1-2.4,ERP2,УНФ1.6,УПП1.3.Частично 1С:7.7ТиС9.2. Самостоятельная разработка правил пользователем для любых документов и конфигураций как в "1С:Конвертации данных 2". Удобная работа с пачкой загруженных документов по пачке сканов. Архив сканов в отдельной базе, без раздувания учетной базы 1С, с отборами по периоду,Организации, Контрагенту и прочим реквизитам извлеченным из сканов. Сохранение пачки отобранных сканов из архива в папку. Переход к загруженному документу 1С по выбранному скану.

Преимущества "Робота"

  1. Цена. В отличие от конкурирующих аналогов, "Робот" ориентирован в первую очередь на получение максимального качества загрузки документов одновременно из нескольких бесплатных способов распознавания. Это позволяет предложить пользователям загрузку неограниченного количества листов документов, за период приобретенной лицензии, по выгодной цене 10800 рублей в год против 4-15 рублей за каждый лист у конкурентов. Загрузка через "Робота" может оказаться в РАЗЫ дешевле, чем через конкурирующие аналоги. 
  2. Локальность. "Робот" может работать без Интернета. Что большая редкость для решений загрузки сканов в 1С. Архив сканов также локальный в базе "Робота".
  3. Удобство. После однократной настройки "Робота" возможен запуск загрузки пачки документов всего одной кнопкой. В учетной базе можно листать сканы загруженной пачки. При этом автоматически отображаются документы загруженные по этой пачке сканов, вместе с распознанными данными. Ни один скан не будет пропущен при обработке загруженных документов.
  4. Поддержка любых графических форматов. Любое изображение редкого формата может быть автоматически преобразовано в JPG (PNG), поддерживаемый всеми способами распознавания. Преобразование выполняется через самый популярный бесплатный консольный редактор ImageMagick идущий в комплекте "Робота".
  5. Поддержка 1С 7.7. На данный момент неизвестно конкурирующих аналогов с поддержкой 1С 7.7.
  6. Универсальность. "Робот" способен частично загружать любые документы без описания правил загрузки документа. Если же надо загружать нестандартный документ(справочник, регистр сведений), то пользователь знакомый с типовой "1С:Конвертацией данных 2", может сам описать любые правила для загрузки любых документов (справочников, регистров сведений) из любых распознанных сканов в любые конфигурации 1С8 и 1С7.7. Также можно описать заполнение нетиповых реквизитов даже не меняя стандартных правил.Едва ли какое-то другое решение на рынке сопоставимо с "Роботом" по возможности поддержки нестандартных входящих форм документов, конфигураций, справочников, регистров и т.п. силами самих пользователей. Кстати самостоятельно разработанные правила можно выгружать в файл и предоставлять другим пользователям "Робота".

Загрузка документов через "Робота"

Сканируем (фотографируем) пачку документов в PDF или отдельные изображения. Если листы документов сканированы в отдельные файлы, то их имена должны сортироваться соответственно. Например, Лист1, Лист2...и т.п.
КАТЕГОРИЧЕСКИ рекомендуется все документы сканировать ШАПКОЙ ВПРАВО. Замечено, что лучше всего бесплатными OCR распознаются сканы шапкой вправо. Также желательно, чтобы сканы были монохромными JPG.
Сохранять сканы можно как в одну папку, так и в разные. В "Роботе" можно или выбирать папку, из которой загружаем документы, или настроить загрузку сразу из нескольких папок.
Если загрузка нужна только в одну базу, то можно настроить загрузку в эту базу по умолчанию. Предоставляется выбор и в какую базу грузим документы.
По умолчанию документы грузятся текущей датой, но можно загрузить выбранной датой или датой входящего документа.
При запуске загрузки документов вся пачка сканов распознается. После распознавания автоматически идентифицируются документы в пачке и их вид. Все листы пачки разбиваются по распознанным документам. Распознанные документы загружаются в базу приемник через COM(OLE). Формируется связка пачка сканов - распознанные данные - документы в 1С.
Документы синхронизируются по реквизитам:
-номер и дата входящего;
-контрагент(ИНН).
Документы записываются не проведенными, без перезаписи существующих документов. Если включено использование архива документов, то сканы соответствующие загруженному документу также сохраняются в базе "Робота".
Автоматически заполняются в документе:
Номер и дата входящего, Организация, Контрагент.
Может заполняться договор (основной или наиболее часто встречающийся за крайний год).
В таблице подставляется "Номенклатура поставщика", Номенклатура, "Единица измерения", Количество, Цена, Ставка НДС, Сумма, НДС, Страна, "Номер ГТД". Справочные типы ищутся по наборам полей. Например, "Номенклатура поставщика" ищется сперва по набору Поставщик+Наименование+Артикул, затем Поставщик+Артикул, затем Поставщик+Наименование.
Учитывается ситуация, когда в скане Количество х Цена НЕ равно Сумма, а есть расхождение в копейки. Для этого выражение Количество х Цена = Сумма сравнивается нечетко, а с точностью до выбранных копеек. По умолчанию в настройках минимальное допустимое расхождение считается 90 копеек. Но можно задать любое. Аналитика (счета учета, статьи затрат и т.п.) заполняются в соответствии с настройками в учетной базе 1С.
Автоматически могут создаваться новые элементы справочников: Контрагенты по ИНН. Номера ГТД.
Полуавтоматически могут создаваться "Номенклатура поставщика", Номенклатура по распознанным данным.

Работы с пачкой документов загруженных через "Робота"

В учетной базе 1С открывается внешняя обработка "Открыть форму дозаполнения", зарегистрированная в базе или из файла.
Эта обработка по умолчанию отображает крайнюю загруженную пачку сканов, но можно выбрать и ранее загруженные.
В обработке можно листать сканы пачки, поворачивать их, автоматически масштабировать на нужный процент для удобства просмотра. При выборе скана отображается документ в 1С соответствующий этому скану.
В результате пользователь может в одном окне видеть пачку сканов, выбранный скан листа, документ соответствующий текущему скану, распознанные и загруженные данные документа.
В этом интерфейсе осуществляется.
     - Сверка сканов, распознанных данных, загруженных документов 1С. Причем данные документа 1С можно видеть не переключаясь на сам документ.
     - Полуавтоматическое создание новой "Номенклатуры поставщика" и "Номенклатуры" по распознанным данным и настройкам пользователя. Пояснение: автоматическое создание новой "Номенклатуры" специально не реализовано, так как "Номенклатура поставщика" и "Номенклатура" часто различаются и в карточке "Номеклатуры" много реквизитов отсутствующих в файлах документов. Кроме пользователя их никто не знает.
     - Обработка документа 1С перед проведением. Например, перенос в накладной строк услуг в таблицу услуг.

Архив документов в "Роботе"

При загрузке сканов в учетную базу 1С, в базе "Робота" автоматически формируется архив сканов. На каждый распознанный документ в базе "Робота" создается "Загруженный документ", к которому прикрепляются сканы всех листов документа. Такая схема позволяет избежать раздувания учетной базы 1С тяжеловесными сканами. Раздувание базы сканами очень нежелательно, так как она будет медленнее работать. По выбранному "Загруженному документу" или выделенному скану можно автоматически открыть документ в учетной базе. В "Загруженном документе" автоматически заполняются поля:
- вид документа;
- номер и дата входящего;
- номер и дата документа в 1С;
- Организация;
- Контрагент;
- Валюта.
По всем этим полям можно отобрать "Загруженные документы" (сканы). Выделенные сканы можно сохранить в папку.

Разработка правил загрузки нестандартных документов в "Роботе"

Конфигурация "Робота" разработана на основе типовой "1С:Конвертации данных 2.1". Все правила загрузки документов, справочников, регистров сведений реализованы прямо в базе. Весь инструментарий разработки правил встроен прямо в сам "Робот".
Разработка правил загрузки данных из сканов максимально унифицирована с обычной разработкой правил обмена между конфигурациями. Распознанные данные сканов выступают в роли виртуальной конфигурации-источника данных, из которой как бы "перегружаются" данные в реальную конфигурацию-приемник.
За загрузку данных на стороне реальной конфигурации-приемника отвечают обычные виды правил 1С:КД2 (ПКД, ПКО, ПКС). За извлечение нужных данных из распознанных отвечают соответствующие им правила "Робота".
"Правилу конвертации данных" (ПКД) реальной конфигурации соответствует "Правило извлечения данных из файла" (ПИДФ) виртуальной конфигурации. "Правилу конвертации объекта" (ПКО) реального объекта соответствует "Правило извлечения объекта"(ПИО) виртуального объекта. "Правилу конвертации свойства"(ПКС) соответствует "Правило извлечения свойства"(ПИС). Обработчикам "Перед загрузкой", "При загрузке", "После загрузки" соответствуют обработчики "Перед извлечением", "При извлечении", "После извлечения". Также есть обработчики идентификации шапки, вида документа, полей поиска. Для всех обработчиков в "Роботе" реализована возможность отладки через внешнюю обработку аналогично отладке обработчиков в 1С:КД2.
Для программиста хорошо знающего 1С:КД2 разработка правил для нестандартного документа вопрос 2-15 часов.
Вы можете заказать разработку правил загрузки любого сканированного документа в любую конфигурацию 1С у нас. (см. раздел техподдержка)
Для разработки правила нужны образцы сканов, конфигурация-приемник, техническое задание на разработку. Правила поставляются в виде файла XML.

Требования к сканам.

- Желательно формат JPG или PNG, можно в файле-контейнере PDF. Если формат другой, то придется настраивать программу для конвертации в JPG или PNG, что замедляет работу. Разрешение 150-300 DPI (точек на дюйм). Желательно монохромное.Размер одного скана листа не более 30 Мб. Желательно не более 1 Мб.
Категорически рекомендуется сканировать все документы строго шапкой вправо.
ПОЯСНЕНИЕ. "Робот" использует много разных способов распознавания. У каждого из OCR свои требования к сканам. В идеале сканы должны соответствовать требованиям всех OCR, что даст в итоге самое высокое качество загрузки сканированных документов.

Код конфигурации "Робота": частично закрыт.
Только одна обработка "УЗД_ОбработкаПравил", ответственная за работу со способами и результатами распознавания, имеет закрытый код.
Код внешних обработок полностью открыт:
- "УЗД_ДозаполнениеПоСкануУФ.epf" для управляемых; "УЗД_ДозаполнениеПоСкануОФ.epf" для обычных форм. 

Лицензирование "Робота"

Параметры лицензии прописаны в файле лицензии: список ИНН организаций, на которые выставлены документы; период действия лицензии; число рабочих мест. В лицензию также включены ключи согласованных API онлайн-сервисов. По умолчанию в лицензию включаются все поддерживаемые бесплатные онлайн-сервисы.
Файл лицензии загружается в базу "Робота". Доступ к архиву документов в "Роботе" неограничен по сроку и списку организаций. Загрузка документов невозможна по истечении периода действия лицензии.

В дистрибутив "Робота" по умолчанию включена лицензия на весь текущий год для типовых демо баз 1С: БП3.0(2.0), УТ10.3-11, КА1.1-2.4, УНФ1.6, УПП1.3 для ИНН 7705260699. Также файл этой лицензии есть в файлах публикации. Вы можете загружать любые документы в сканах, которых есть ИНН организации 7705260699. Никаких ограничений по функционалу для бесплатной, тестовой лицензии нет.
В файлах публикации есть образцы сканированных документов с ИНН 7705260699 для загрузки в демо-базы 1С. Также Вам может быть предоставлена бесплатная индивидуальная демо-версия на Вашу организацию сроком на 3 месяца.

Техническая поддержка

Техподдержка действует на период времени действия приобретенной лицензии. Можно обращаться по электронной почте, также возможно подключение по удаленке через AnyDesk и AmmyyAdmin для начальной установки и настройки "Робота".

Обновления

Обновления полностью бесплатны.Обновления обязательные для корректной загрузки регламентированных документов (УПД,СФ,ТОРГ12) отмечаются особо. Пользователи будут отдельно извещаться об таких обновлениях. И пользователям надлежит их отслеживать. (На практике такие обновления редки, может быть менее 1-2 раза в год)

Вопросы возврата

Так как "Робот" может работать локально, не существует возможности проконтролировать прекращение использования "Робота". Поэтому пользователям предлагается получить бесплатную индивидуальную демоверсию на 3 месяца, чтобы убедиться что "Робот" подходит Вашей организации.
